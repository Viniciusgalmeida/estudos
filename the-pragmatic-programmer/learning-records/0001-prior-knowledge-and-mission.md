# Prior knowledge and mission

User is a developer (not yet senior) reading The Pragmatic Programmer chapter by chapter. Goal: internalize the mindsets that distinguish senior engineers. Same overarching goal as the software-architecture study track.

**Implication:** Frame tips through the lens of daily coding decisions, not abstract career advice. When teaching a concept, connect it to something the user would actually encounter in a PR, a code review, or a technical discussion.
