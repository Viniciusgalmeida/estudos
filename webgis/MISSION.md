# Mission: WebGIS

## Why
Starting a front-end developer role that involves displaying and interacting with satellite imagery. Need to become productive quickly with WebGIS concepts and tooling — specifically building map UIs in React that let users view satellite photos, draw/annotate on top of them, and visualize data layers.

## Success looks like
- Understand how web maps work at a level that lets you debug problems confidently
- Render a satellite map in a React app using MapLibre GL JS
- Add interactive drawing/annotation tools over a map
- Display data layers (GeoJSON shapes, heatmaps, etc.) on top of satellite imagery
- Speak the language of the team: tiles, projections, layers, sources, GeoJSON

## Constraints
- React developer (comfortable with JS/JSX, component model)
- Some prior exposure to WebGIS concepts but not deep
- Library may be MapLibre GL JS (not confirmed yet)
- Learning alongside starting a new job — lessons should be short and directly applicable

## Out of scope
- Server-side GIS (PostGIS, GDAL, geoprocessing pipelines)
- Desktop GIS tools (QGIS, ArcGIS)
- Cartography theory beyond what's needed to work with web maps
