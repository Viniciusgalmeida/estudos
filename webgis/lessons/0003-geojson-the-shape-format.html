GeoJSON:
The Shape Format

What GeoJSON is

GeoJSON is a JSON format for describing geographic shapes and their metadata. It is the universal language of web mapping: drawing tools produce it, APIs return it, and MapLibre's geojson source consumes it directly.

You do not need a special library to work with it. It is just JSON with a well-defined structure. If you can write a JavaScript object, you can write GeoJSON.

Retain this GeoJSON has three levels: Geometry (the raw shape), Feature (shape + metadata), and FeatureCollection (an array of features). MapLibre always expects a FeatureCollection.

The three levels

FeatureCollection
The top-level object. Contains an array of Features. This is what you pass to addSource and setData.
Feature
One item in the collection. Wraps a Geometry and adds a properties object for metadata (name, area, crop type…).
Geometry
The actual shape: Point, LineString, or Polygon (plus their Multi- variants). Contains only coordinates.
Properties
Any key-value metadata you want to attach: { "name": "Field A", "area_ha": 12.4 }. MapLibre can use these to style layers conditionally.

Geometry types

Point
Sample location, marker, sensor
LineString
Path, transect, road
Polygon
Field boundary, annotation area

There are also MultiPoint, MultiLineString, and MultiPolygon for cases where one Feature needs to represent several disconnected shapes of the same type — for example, a farm split into two separate plots.

The coordinate gotcha

Common mistake GeoJSON coordinates are [longitude, latitude] — in that order. This is the opposite of how most people say it ("lat, lng"). MapLibre also expects [lng, lat]. If your shapes appear in the wrong place or at sea, check coordinate order first.

Memory trick: GeoJSON follows the mathematical convention of (x, y) — and on a map, longitude is the horizontal axis (x) and latitude is vertical (y). So: x = lng = first. Always.

What the JSON looks like

Here is a complete, minimal FeatureCollection with one polygon — a field boundary:

// A complete GeoJSON FeatureCollection { "type": "FeatureCollection", "features": [ { "type": "Feature", "geometry": { "type": "Polygon", "coordinates": [ // outer ring — array of [lng, lat] positions [ [-46.640, -23.548], [-46.638, -23.548], [-46.638, -23.550], [-46.640, -23.550], [-46.640, -23.548] // must close: repeat first point ] ] }, "properties": { "name": "Field A", "crop": "soy", "area_ha": 12.4 } } ] }

Two things to notice: the polygon's outer ring is an array wrapped in another array (because a Polygon can have holes — more on that later). And the ring closes by repeating the first coordinate at the end.

Working with it in JavaScript

In practice you will build GeoJSON objects from user interactions or API responses, then push them into MapLibre. Here is the full pattern from lesson 2, shown with real GeoJSON:

// Build a GeoJSON feature from coordinates the user drew function buildFieldFeature(lngLatRing, properties) { const closed = [...lngLatRing, lngLatRing[0]]; // close the ring return { type: 'Feature', geometry: { type: 'Polygon', coordinates: [closed] }, properties: properties }; } // Build a FeatureCollection from an array of features function toCollection(features) { return { type: 'FeatureCollection', features: features }; } // Push into MapLibre — same pattern as lesson 2 function renderFields(map, fields) { const features = fields.map(function(field) { return buildFieldFeature(field.ring, { name: field.name, crop: field.crop }); }); map.getSource('annotations').setData(toCollection(features)); }
