Sources & Layers:
How MapLibre Structures a Map

Two concepts, always together

Every map in MapLibre is built from two types of objects: sources and layers. They are always separate, and they always work together.

A source answers the question: where does the data come from?
A layer answers the question: how should that data be drawn?

Retain this Sources hold data. Layers draw it. You never draw directly from a source — you always go through a layer. One source can feed multiple layers.

The map as a stack of layers

Think of your map as a stack of transparencies, rendered from bottom to top. The satellite image sits at the bottom. Everything else — field boundaries, annotations, labels — floats on top of it, in the order you define.

Example stack for a satellite + field annotation map
layer field-labels type: symbol  ·  text on top
layer field-fill type: fill  ·  yellow polygon
layer field-outline type: line  ·  white border
layer satellite type: raster  ·  the photo tiles
↑   rendered last (on top)
↓   rendered first (at bottom)

Notice: field-fill, field-outline, and field-labels could all share the same source — one geojson source containing the field polygons. Three layers, one source. This is intentional — it means you can style the same data multiple ways without duplicating it.

Source types you will use

Type What it holds Your job context
raster PNG/JPG tile URL template satellite imagery
geojson GeoJSON object or URL annotations & drawings
vector Vector tile URL template Base map roads, labels
image One georeferenced image Overlaying a single photo

In code: satellite map from scratch

Here is the minimum code to show a satellite map in MapLibre. Read each part — it maps directly onto the concepts above.

import maplibregl from 'maplibre-gl'; function initMap(containerId, tileUrl) { const map = new maplibregl.Map({ container: containerId, style: { version: 8, sources: {}, // empty — we'll add below layers: [] // empty — we'll add below }, center: [-46.63, -23.55], // [lng, lat] zoom: 12 }); map.on('load', function() { addSatelliteLayer(map, tileUrl); }); return map; } function addSatelliteLayer(map, tileUrl) { // 1. Register the source — WHERE the data comes from map.addSource('satellite', { type: 'raster', tiles: [tileUrl], // e.g. 'https://tiles.example.com/{z}/{x}/{y}.jpg' tileSize: 256 }); // 2. Add a layer — HOW to draw that source map.addLayer({ id: 'satellite-layer', // unique name for this layer type: 'raster', // must match source type for raster source: 'satellite' // references the source by its id }); }
MapLibre expects coordinates as [longitude, latitude] — note the order. This trips up almost everyone at first.

Adding a GeoJSON layer on top

Now add drawn annotations — a separate source and layer, stacked above the satellite. The satellite layer stays untouched.

function addAnnotationLayer(map) { // Source: a GeoJSON collection (starts empty) map.addSource('annotations', { type: 'geojson', data: { type: 'FeatureCollection', features: [] } }); // Layer: draw the polygons as filled shapes map.addLayer({ id: 'annotations-fill', type: 'fill', source: 'annotations', paint: { 'fill-color': '#ffff00', 'fill-opacity': 0.3 } }); // Another layer — same source, draws the border map.addLayer({ id: 'annotations-outline', type: 'line', source: 'annotations', paint: { 'line-color': '#ffffff', 'line-width': 2 } }); } // Later, when the user draws a polygon, update the source: function updateAnnotations(map, newFeatures) { map.getSource('annotations').setData({ type: 'FeatureCollection', features: newFeatures }); // Both layers update automatically — the source is the single source of truth }

Notice updateAnnotations: you update the source, not each layer individually. Both annotations-fill and annotations-outline re-render automatically because they share the source. This is the power of the source/layer split.

The pattern you will use constantly When new data arrives (user draws a shape, API returns field boundaries), call getSource(id).setData(newGeoJSON). The layers take care of themselves.
